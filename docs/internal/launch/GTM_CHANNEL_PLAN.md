# GTM Channel Plan — Circle MCP Server

> **Version:** 1.0.0
> **Last Updated:** 2026-03-13
> **Purpose:** Define where and how to distribute Circle MCP for early launch traction

---

## Channel Strategy Philosophy

This is a founder-led launch of a premium technical product. The GTM is:
- **High-signal, low-volume** — 30–50 targeted outreach messages beat 1,000 generic ones
- **Proof-first** — every channel interaction should include evidence (live endpoint, GitHub repo, demo)
- **Relationship-driven** — the first 10 clients come from conversations, not funnels
- **Dual-track** — services revenue from direct outreach + product awareness from distribution channels

---

## Priority Channels

### Tier 1 — High Priority (Week 1–2)

#### 1. LinkedIn (Direct Outreach + Content)

| Attribute | Detail |
|-----------|--------|
| **Why** | Circle consultants, community strategists, and agencies are active on LinkedIn. Professional context matches the buyer persona. |
| **Message fit** | Professional, value-led, proof-backed. LinkedIn tolerates longer posts with technical depth. |
| **Audience quality** | High — professionals who match the ICP are identifiable and reachable |
| **Effort** | Low-Medium — 1 launch post + 10–15 targeted DMs per week |
| **Expected outcome** | 3–8 conversations from 15 DMs, 1–2 demo calls per week |

**Actions:**
1. Post launch announcement (see LAUNCH_ANNOUNCEMENTS.md)
2. Send personalized DMs to Circle consultants and agency leads
3. Engage with Circle.so-related posts and threads to build visibility
4. Post follow-up content in Week 2–3 (client learnings, use cases)

#### 2. GitHub (Repo + Discussions)

| Attribute | Detail |
|-----------|--------|
| **Why** | Technical operators and MCP ecosystem builders discover tools through GitHub. The repo is the product's credibility anchor. |
| **Message fit** | Technical, documentation-backed, transparent about capabilities and limitations |
| **Audience quality** | High for technical operators; medium for service buyers |
| **Effort** | Low — repo is ready, post Discussion announcement, respond to issues |
| **Expected outcome** | 10–20 stars in 30 days, 2–5 issues/discussions, some self-serve users |

**Actions:**
1. Post GitHub Discussions announcement
2. Ensure README is discoverable (keywords, description, topics)
3. Respond to every issue and discussion within 24 hours
4. Tag with relevant topics: `mcp`, `circle-so`, `community`, `ai`, `claude`

#### 3. Smithery Marketplace

| Attribute | Detail |
|-----------|--------|
| **Why** | MCP-specific discovery channel. Users browsing Smithery are already MCP-aware — zero education cost. |
| **Message fit** | Standard MCP listing with clear description and config schema |
| **Audience quality** | High for technical MCP users; variable for service conversion |
| **Effort** | Very Low — publish listing (config is ready), monitor |
| **Expected outcome** | Passive discovery, 5–15 installs in first month |

**Actions:**
1. Submit Smithery listing (config complete, see SMITHERY_PUBLISHING_PACKAGE.md)
2. Verify listing renders correctly
3. Monitor install count weekly

### Tier 2 — Medium Priority (Week 2–3)

#### 4. Targeted Outreach (Email)

| Attribute | Detail |
|-----------|--------|
| **Why** | Email reaches buyers who are not active on LinkedIn or GitHub. Longer-form messaging works for agencies and educators. |
| **Message fit** | Professional, personalized, value-specific to the recipient's business |
| **Audience quality** | High — hand-selected recipients based on public profiles and community activity |
| **Effort** | Medium — requires list-building and personalization |
| **Expected outcome** | 10–20% reply rate on personalized emails, 2–4 demo calls from 20 emails |

**Actions:**
1. Build list of 20–30 targets: Circle consultants, agency leads, active educators
2. Send segment-specific emails (see OUTREACH_COPY.md)
3. Follow up at 5–7 days
4. Track in simple spreadsheet

#### 5. Circle Expert Networks / Communities

| Attribute | Detail |
|-----------|--------|
| **Why** | Circle experts (community managers, Circle-certified consultants) are the most qualified buyers. They are concentrated in specific communities and forums. |
| **Message fit** | Peer-level, value-first, offer-to-demo |
| **Audience quality** | Very High — these people manage Circle communities professionally |
| **Effort** | Medium — requires identifying and joining relevant communities |
| **Expected outcome** | 2–5 high-quality conversations |

**Actions:**
1. Identify Circle expert communities (Circle's own community, Circle consultant networks)
2. Post concise community announcement (see LAUNCH_ANNOUNCEMENTS.md)
3. Engage genuinely — answer questions, share insights, offer demos
4. Do NOT spam — one well-crafted post is better than daily promotion

#### 6. Direct Demos

| Attribute | Detail |
|-----------|--------|
| **Why** | A live demo converts better than any written copy. Showing the system working against a real community creates immediate buy-in. |
| **Message fit** | Interactive, tailored to the buyer's community and use case |
| **Audience quality** | Highest — only people who schedule demos are genuinely interested |
| **Effort** | High per interaction (15–30 min each), but highest conversion |
| **Expected outcome** | 50–70% conversion from demo to paid engagement |

**Actions:**
1. Prepare a repeatable 15-minute demo flow (health check → unanswered posts → create a draft post)
2. Offer demos as the CTA in all outreach
3. End every demo with a clear next step (offer, timeline, pricing)

### Tier 3 — Low Priority / Defer (Week 4+)

#### 7. X / Twitter

| Attribute | Detail |
|-----------|--------|
| **Why** | AI/developer community is active. Good for visibility and credibility, but lower conversion than LinkedIn for this buyer. |
| **Message fit** | Concise, technical, proof-oriented |
| **Audience quality** | Medium — broad developer audience, not all relevant |
| **Effort** | Low — 1 launch post + occasional engagement |
| **Expected outcome** | Visibility, some GitHub traffic, unlikely to drive service revenue directly |

**Action:** Post launch tweet + thread. Engage with MCP-related conversations. Do not invest heavily.

#### 8. Partner / Referral Channel

| Attribute | Detail |
|-----------|--------|
| **Why** | Circle consultants who adopt the tool become referral sources for other consultants and agencies. |
| **Message fit** | Relationship-based, mutual value proposition |
| **Audience quality** | Very High (referred leads) |
| **Effort** | Low initially — emerges naturally from good first-client experiences |
| **Expected outcome** | 1–3 referrals per happy client over time |

**Action:** After successful engagements, ask "Do you know other Circle professionals who would find this useful?" Do not formalize a referral program yet.

## Channels to Defer

| Channel | Why Defer |
|---------|-----------|
| **Product Hunt** | Product is not consumer-grade self-serve. PH audience expects polished onboarding. Defer to v2/v3. |
| **Paid advertising** | No landing page, no funnel, no scale need. Premature at 0 clients. |
| **Podcast / media appearances** | Need a track record and client stories first. Revisit after 10+ engagements. |
| **Content marketing / blog** | LinkedIn posts are sufficient for v1. A blog adds maintenance without proportional reach. |
| **Circle.so partnership** | Need proven adoption first. 5+ paying clients and testimonials required before approaching Circle's team. |
| **Cold calling** | Mismatched with premium positioning. Warm outreach only. |
| **Community building (own Discord/Circle)** | Premature. You are one person. Engage in existing communities instead. |
